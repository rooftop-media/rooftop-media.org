Software is our specialty. We provide a set of open-source creative productivity apps, like editors for scripts, photos, video, wikis, financial sheets, web pages, publishable print designs, & more.

    Below are two sets of goals we have for our software -- our goals for the completed software apps, and our goals for the internal design of the code for those apps.


Our software apps should ...

...be easy to understand.

...be efficient to use.

...serve a practical use.

...be focused and well-organized.

Our software code should ...

...have few external dependencies.

...be built with known patterns.

...be well documented & justified.

...be customizable.


    In order to produce software that meets those goals, we ended up researching & designing a full software stack, built on our own framework, with nothing but vanilla NodeJS.


    We've organized our software efforts into 3 layers:

  • the Applications, which are useful productivity programs.
  • the Framework, which are tools used to make those applications.
  • the Research, which describes underlying software details & principles.

    Each of those layers is described by 3 books, creating a sort of trilogy-of-trilogies.

    The project, to make these 9 guidebooks and the software that accompanies them, is called KitchenWare. The name is inspired by the fine control some experienced chefs have, when equipped with the right kitchen equipment.

The Framework


Making Ktty

Ktty is KitchenWare's first foundational program.
Ktty is actually 4 programs in one, all made specially for the text-terminal.

The four programs are:
  • A plain text editor.
  • A code editor, with code editor niceties.
  • A web browser, which renders html/css.
  • A web server, which serves html/css.

The Ktty code tutorial is also on Github.
Ktty is all about making it clear how these programs are built.
The guidebook, "Making Ktty", explains the code step-by-step, in vanilla NodeJS.

The Ktty system is a "What You See Is What You Get" system (WYSIWYG), inspired by the classic Model-View-Controller architecture.
Such terms are part of the classic object-oriented patterns developed in the Smalltalk operating system.

This project was inspired, in part, by the 1991 book titled "Inside Smalltalk."
Also, I cite "Design Patterns: Elements of Reusable Object-Oriented Code", from 1995.




Making Cookbook

Cookbook is a component-based framework for making website page designs.
The framework is inspired by the features of other popular web-frameworks, like VueJS, Angular, or React.

Cookbook is for designing professional website pages. Features include:
  • An object-oriented component system!
  • Single file components - html, css, & js in one.
  • Use reactive data in html & css.
  • Page routing organization options.
We'll use CookbookJS to make all our app interfaces. So it's pretty important!
The guidebook, "Making Cookbook," describes how to write & use Cookbook, for Ktty's web browser as well as standard, GUI browsers, like Chrome or Firefox.
